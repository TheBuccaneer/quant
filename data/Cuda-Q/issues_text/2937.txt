### Describe the bug

For my specific circuit shown in the code below, there should be 16 items in the count result, but there are only 8 items with the tensornet or tensornet-mps target. With the nvidia target, I can get the correct simulation result. I know the tensornet target outputs a sample result, but it seems that the result is far from the correct one. Please let me know if this issuce could be fixed by some simple configuration.

### Steps to reproduce the bug

The Python code to reproduce the bug:

import cudaq
import numpy as np

qubit_count = 20

cudaq.set_target("tensornet")

kernel = cudaq.make_kernel()

qubits = kernel.qalloc(qubit_count)


depth = qubit_count // 5
for i in range(0, depth):
    kernel.h(qubits[i * 5])
    for j in range(4):
        kernel.cx(qubits[i * 5+j], qubits[i * 5+j+1])

kernel.mz(qubits)

result = cudaq.sample(kernel, shots_count=10240)
result.dump()
